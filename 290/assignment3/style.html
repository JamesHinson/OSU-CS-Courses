Styles

Locations

CSS style rules can be applied in different locations within an HTML document or in an external CSS file. Styles can be directly applied to HTML elements using the style attribute, known as inline styles. Alternatively, style rules can be included within the <style> element in the <head> section of an HTML document, called internal styles. Another method is to store style rules in an external CSS file and link it to the HTML document using the <link> element in the <head> section, referred to as an external stylesheet. Additionally, CSS rules can be imported into another CSS file using the @import rule for modularization. Browsers have default stylesheets applied to HTML elements, and users can apply their own custom styles through browser extensions or settings. Understanding these locations allows for effective organization and management of CSS styles in web development.

Selectors

Selectors in CSS are powerful tools used to target specific elements within an HTML document for styling. Class selectors, denoted by a dot (.) followed by the class name, allow styling elements that share a common class attribute. ID selectors, indicated by a hash (#) followed by the ID name, target unique elements with a specific ID attribute. Attribute selectors let you style elements based on their attribute values, providing flexibility in targeting elements with certain attributes or values. Pseudo-classes and pseudo-elements, prefixed by a colon (:), enable styling based on element states or positions within the document, such as :hover for mouse hover effects or ::before to insert content before an element. By utilizing these different types of selectors, developers can precisely define which elements to style and apply customized designs to their web pages.

Box Model

The box model is a fundamental concept in CSS that describes the layout and structure of elements on a web page. According to the box model, every element in HTML is treated as a rectangular box, consisting of content, padding, borders, and margins. The content area contains the actual content of the element, such as text or images. Padding is the space between the content and the element's border, providing internal spacing. Borders surround the padding and content, defining the visible edges of the element. Margins are the space outside the element's border, creating separation between adjacent elements. Understanding the box model is crucial for designing layouts, as it determines how elements are positioned and interact with each other on the page, influencing spacing, sizing, and overall visual presentation.

Inheritance & Specificity

In CSS, inheritance and specificity are two key concepts that govern how styles are applied to elements in a document. Inheritance refers to the mechanism by which certain properties of a parent element are passed down to its children. This means that if a style property is applied to a parent element, its child elements will inherit that property unless specifically overridden. Specificity, on the other hand, determines which style rule takes precedence when multiple conflicting rules are applied to the same element. CSS rules are assigned a specificity value based on the selectors they contain, with more specific selectors taking precedence over less specific ones. In essence, inheritance establishes a hierarchical relationship between elements, while specificity resolves conflicts between competing style rules, ensuring that the correct styles are applied according to the document structure and the specificity of the rules.
